3. ARCHITECTURAL DIRECTIVES (THE "ZERO-ANOMALY" RULES)

3.1 Atomic Storage Pipeline (The "Vault" Rule)
Any file write to the persistent storage must follow this exact sequence to survive a power pull at any microsecond:
1. Stage: Write to /mnt/ram_shield (tmpfs).
2. Process: Compress $\to$ Encrypt $\to$ Hash (SHA-256).
3. Pending: Write to both USBs as .pending files.
4. Hardware Sync: Call fsync() on the file handles.
5. Commit: Atomic rename() to .sealed.
6. Directory Sync: Call sync_all() on the parent directory to lock the filename change into the hardware.
3.2 Memory & Sandbox Budget (8GB Pi 5 Ceiling)
To avoid Out-Of-Memory (OOM) panics, the following limits are non-negotiable:
* RAM Shield (tmpfs): 1GB (Capped at /mnt/ram_shield).
* AI Sandbox (cgroup): 4GB Max (MemoryMax=4G).
* Supervisor/OS/Dashboard: 3GB Reserved.
3.3 The IPC Airgap (Root vs. User)
* The Rust supervisor (Root) binds /tmp/mission.sock.
* It must execute chmod 0o666 on the socket immediately after binding.
* The Python Sandbox (User) connects to provide inference results.
* Timeout: Rust must wrap UDS reads in a 2500ms timeout (extending to 20s during low-power mode).

4. ALGORITHMIC SCHEMAS
4.1 Priority Scoring ($P_{Score}$)
The transmission queue is a Binary Heap sorted by:


$$P_{Score} = (W_c \cdot C) + (W_a \cdot T_{wait}) - (W_s \cdot S_{mb})$$
* C (Criticality): 1000 for Anomaly, 1 for Routine.
* T (Wait): Seconds elapsed since ingest.
* S (Size): Payload size in MB.
4.2 Power Governor Logic
* State 1 (Solar): SoC $> 50\% \to$ AI CPUQuota=200%.
* State 2 (Eclipse): SoC $< 30\% \to$ AI CPUQuota=5%.
* Mechanism: Rust invokes systemctl set-property mission-ai.service CPUQuota=X% via DBus.
